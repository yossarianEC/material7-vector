Vector Shotlist System SOP

Generate the blue internal shotlist first, then derive the green client/talent guide from the same structured campaign data.

Required Input

Minimum useful data

Client name, shoot date, location, start time, and campaign/product/service context. Add confirmed claims, do-not-say rules, preferred CTAs, and product lists when available.

Claim Safety

No invented promises

Do not invent free, cheapest, best-in-market, guaranteed outcomes, discounts, promotions, specs, or stock availability.

Production Logic

Shoot by block

Capture all intros, then all benefits, then all CTAs, then b-roll. This reduces resets, talent movement, confusion, and editing friction.

Publishing

Clean studio URLs

Save generated HTML in /shotlists/. The public URL is https://studio.material7.com/shotlists/file-name.html.

Canonical Files

Do not recreate from memory

Blue template /shotlist-system/templates/blue.html
Green template /shotlist-system/templates/green.html
Master CSS /shotlist-system/styles/material7-shotlist.css
